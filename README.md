# منظومة تقييم العائلات — مبرة عطاء

تطبيق Laravel 12 لرقمنة استمارة تقييم العائلات المحتاجة وأتمتة محرّك حساب النقاط والتوصية بالاستحقاق.

> القرارات المعمارية الكاملة في [CLAUDE.md](CLAUDE.md).

## التشغيل المحمول من فلاشة USB

البرنامج محمول بالكامل: قاعدة البيانات SQLite (ملف واحد)، والأصول مبنية مسبقاً، و`vendor` مرفوع في المستودع — فلا حاجة لـ XAMPP ولا Composer ولا Node على جهاز التشغيل. المطلوب فقط **PHP 8.2+ محمول** بجانب المشروع.

### بنية الفلاشة
```
USB:\
├── charity-assessment\   ← هذا المستودع
└── php\php.exe           ← PHP 8.2/8.3 Windows x64 Thread Safe (ZIP، بلا تثبيت)
```
فعّل في `php\php.ini` الإضافات: `pdo_sqlite`, `sqlite3`, `gd`, `fileinfo`, `mbstring`, `openssl`.

### الملفات الجاهزة (نقرة مزدوجة) — أسماء ومحتوى إنجليزي عمداً (راجع CLAUDE.md لسبب تفادي أي عربي داخل bat)
| الملف | الوظيفة |
|------|---------|
| `first-install.bat` | **مرة واحدة بعد الاستنساخ:** ينشئ `.env` + مفتاح + قاعدة بيانات فارغة + يبني الجداول |
| `run.bat` | تشغيل الخادم وفتح المتصفح على `http://127.0.0.1:8000` |
| `update.bat` | جلب آخر نسخة من GitHub (`git pull`) + ترحيل + تنظيف الكاش (يأخذ نسخة احتياطية أولاً) |
| `backup.bat` | نسخ `database.sqlite` إلى `backups\` بطابع زمني (يُبقي آخر 30) |

## التثبيت الأول على جهاز جديد
```bash
git clone <عنوان-المستودع> charity-assessment
```
ثم نقرة مزدوجة على **`first-install.bat`** ← ثم **`run.bat`**.
بعد أول تشغيل: سجّل المستخدم الأول، افتح «سياسة النقاط» واحفظ لإنشاء **السياسة الأولى v1**، ثم ابدأ إدخال العائلات.

## التحديث لاحقاً
نقرة مزدوجة على **`update.bat`** فقط. **بياناتك المحلية (`database.sqlite` و`.env`) لا تتأثر** لأنها مستثناة من المستودع.

## ملاحظات مهمة
- **البيانات محلية:** `database/*.sqlite` و`.env` والمرفقات والنسخ الاحتياطية **غير مرفوعة** على GitHub — تبقى على الفلاشة فقط.
- خذ نسخة احتياطية دورية وانقل مجلد `backups\` إلى قرص آخر.
- لا تنزع الفلاشة ونافذة الخادم مفتوحة.

## للمطوّر
بعد تعديل Blade/CSS: `npm run build` ثم **commit لملفات `public/build`** (لأنها مرفوعة).
بعد تحديث الاعتماديات: `composer install` ثم **commit لمجلد `vendor`**.
بهذا يصل التحديث جاهزاً لمستخدم الفلاشة دون أن يحتاج Composer/Node.
